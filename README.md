# CCIF — Criminal Intelligence Fabric

CCIF is a full-stack criminal-intelligence dashboard for managing cases, suspects, alerts, evidence, crime domains, and their relationships.

The project consists of:

- `ccif-frontend/` — React + Vite interface using Cytoscape for the intelligence-graph visualization.
- `backened/` — FastAPI API backed by PostgreSQL.

## Current capabilities

- View and create case records.
- View suspects, alerts, evidence, officers, dashboard metrics, and crime domains.
- Visualize cases, suspects, evidence, gangs, and locations as an interactive relationship graph.
- Store locally created cases, suspects, and graph additions in browser local storage when the API is unavailable.

## Intelligence graph

The graph is generated by the backend from PostgreSQL records. Relationships are derived using predefined rules, such as shared locations and suspect gang membership. It is displayed with Cytoscape.

This version does **not** include Neo4j, machine-learning models, or an active AI/LLM integration.

## Requirements

- Node.js and npm
- Python 3.10+
- PostgreSQL with a `ccif` database and the tables used by the API: `cases`, `suspects`, `alerts`, and `evidence`

## Run locally

### 1. Configure PostgreSQL

Update the connection settings in `backened/app/database.py` to match your local PostgreSQL instance. Do not commit real database credentials.

### 2. Start the backend

```bash
cd backened
python -m venv .venv
```

Activate the virtual environment, then install dependencies and run FastAPI:

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

The API is available at `http://localhost:8000`; interactive docs are at `http://localhost:8000/docs`.

### 3. Start the frontend

In a second terminal:

```bash
cd ccif-frontend
npm install
npm run dev
```

Open the local Vite address shown in the terminal (normally `http://localhost:5173`). During development, Vite proxies `/api` requests to the backend on port 8000.

## Main API routes

| Route | Purpose |
| --- | --- |
| `GET/POST /cases/` | List or create cases |
| `GET /cases/{case_id}` | Get a case |
| `GET /suspects/` | List suspects |
| `GET /alerts/` | List alerts |
| `GET/POST /evidence/` | List or add evidence |
| `GET /evidence/{case_id}` | Get evidence for a case |
| `GET /dashboard/stats` | Dashboard metrics and recent activity |
| `GET /graph/network` | Nodes and edges for the intelligence graph |
| `GET /crimes/` | Crime-domain metadata |
| `GET /officers/` | Officer metadata |

All routes are also exposed beneath the `/api` prefix for the frontend proxy.

## Optional seed data

To add sample evidence after PostgreSQL is configured:

```bash
cd backened
python -m app.seed_evidence
```
